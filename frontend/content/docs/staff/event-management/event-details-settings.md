After creating an event, the details page is where you fill in everything visitors and exhibitors need to know: dates, venue, description, and configuration options.

## What you will learn

- How to edit event details
- What each setting controls

## Editing event details

1. Open your project, then click the event
2. Go to **Details** (or click the event name)
3. Update the fields you need and click **Save**

### Fields

- **Name** - the full event title (e.g., "Megabuild Indonesia 2026")
- **Slug** - URL identifier, used in API paths
- **Date range** - start and end dates of the event
- **Venue** - where the event takes place
- **Description** - a rich text overview shown on the event website
- **Logo and banner** - images used on the website and in admin views

## Event settings

The **Settings** tab within an event controls:

- **Event visibility** - whether the event website shows this event publicly
- **Registration settings** - controls for attendee registration if applicable
- **Contact form** - which form receives submissions from the event website's contact page

## Common questions

**Q: Can I schedule an event to go live on a specific date?**

There is no scheduled publishing. You control visibility manually through the event settings toggle.

**Q: Where do event details appear?**

On the event website's homepage, about page, and anywhere the API returns event metadata (sitemap, SEO tags, etc.).
