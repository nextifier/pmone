Events live inside projects. Each event represents a specific exhibition edition (e.g., "Megabuild Indonesia 2026") and gets its own set of brands, content, orders, and documents.

## What you will learn

- How to create an event
- How to set an event as active
- How to reorder, delete, and restore events

## Creating an event

1. Open a project from the **Projects** list
2. Go to the **Events** tab
3. Click **Create**
4. Fill in the event name, slug, and date range
5. Click **Save**

The slug appears in API URLs and on your event website. Keep it short: `2026`, `march-2026`, or similar.

## Setting the active event

Each project can have one active event at a time. The active event is the one your event website serves by default.

To change it: open the event and click **Set as active**, or use the toggle in the event list.

## Reordering events

Drag and drop events in the list to change their display order. This affects the order in which past editions appear in the admin panel.

## Deleting and restoring

Delete an event through the three-dot menu. It moves to the trash. Restore it from **Events > Trash** if you change your mind.

Permanent deletion is available from the trash view. This removes the event and all its associated data (brands, orders, documents) for good.

## Common questions

**Q: Can I duplicate an event to reuse last year's setup?**

There is no one-click duplication. You will need to create a new event and re-import brands and products using the import feature.

**Q: What does "active event" affect?**

The event website uses the active event to serve its content. Only one event per project can be active. Visitors see data from whichever event is marked active.
